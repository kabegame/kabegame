Kabegame 在桌面端启动后会在系统托盘放置一个图标，作为隐藏窗口后重新呼出主界面、快速切换壁纸、彻底退出应用的入口。该功能仅在桌面端（Windows / macOS / Linux）可用。

## 托盘位置

Kabegame 会在系统托盘（Windows 任务栏右侧、macOS 菜单栏右侧、Linux 通知区）显示一个图标。鼠标悬停时会显示 Kabegame 提示文本。

托盘图标是**静态**的，不会随下载、轮播等状态变化：

- macOS 使用模板图，会自动跟随菜单栏的深浅主题反色
- Windows / Linux 沿用应用主窗口图标

:::note
应用启动后约 500 毫秒内托盘图标尚未创建，这段时间内左键点击与菜单都不会响应。
:::

---

## 左键点击

左键点击托盘图标用于切换主窗口的显隐：

- 窗口当前**隐藏**或已被销毁 → 显示主窗口并获取焦点；若窗口不存在则会重建
- 窗口当前**显示** → 隐藏主窗口（应用仍在后台运行，壁纸轮播不受影响）

左键点击带有 500 毫秒防抖，过快的连续点击会被忽略。

:::note
左键点击不会弹出菜单。菜单只能通过右键点击打开。
:::

---

## 右键点击

右键点击托盘图标会显示上下文菜单，按从上到下顺序包含以下四项：

| 菜单项 | 说明 |
|--------|------|
| 显示窗口 | 显示主窗口并获取焦点；若主窗口已被销毁会重建一个新的主窗口 |
| 隐藏窗口 | 隐藏主窗口。**不会**影响壁纸窗口和爬虫子窗口 |
| 下一张壁纸 | 立即切换到下一张壁纸。依赖已在设置中选择画册，否则切换失败 |
| 退出 | 立即退出应用，**无二次确认**；进行中的下载或整理任务会被中断 |

:::caution
「退出」不会弹出确认对话框，也不会等待后台任务结束。如果正在下载或整理图片，请先在主窗口停止任务，再从托盘退出。
:::

---

## 语言切换

应用语言在设置页切换后，托盘菜单文本与 tooltip 会自动刷新，**无需重启**。托盘文本与前端界面使用两套独立的国际化文件，但切换时机完全同步。

:::note
目前没有用于显示 / 隐藏托盘图标的设置项，托盘在桌面端始终启用。
:::

---

## 关闭窗口 ≠ 退出

点击主窗口标题栏的「×」时，应用默认**隐藏窗口**而不是退出，这样可以让壁纸轮播等后台功能继续工作。

真正要退出应用，只能通过托盘菜单中的「退出」。`Alt+F4` 同样会被拦截为隐藏。

:::note
如果关窗后找不到主窗口，可以左键点击托盘图标或从右键菜单选「显示窗口」恢复。
:::

## 平台差异

| 能力 | Windows | macOS | Linux | Android |
|---|:---:|:---:|:---:|:---:|
| 托盘图标显示 | ✅ | ✅ | ⚠️ | ❌ |
| 模板图自动反色 | ❌ | ✅ | ❌ | — |
| 左键切换窗口 / 右键菜单 | ✅ | ✅ | ✅ | ❌ |
| 关闭主窗口转为隐藏 | ✅ | ✅ | ✅ | — |

### Linux：AppIndicator / StatusNotifierItem

Linux 下的托盘依赖 **AppIndicator / Ayatana AppIndicator**（StatusNotifierItem 协议）。如果桌面环境不支持 SNI，**托盘图标不会出现**，并且当前版本没有回退到通知或其它 UI。

- Debian / Ubuntu 系发行版需要安装 `libayatana-appindicator3-1`
- **GNOME**（默认）需要额外启用「AppIndicator and KStatusNotifierItem Support」扩展，否则不显示图标
- **KDE Plasma**、**XFCE**（含 `xfce4-statusnotifier-plugin`）、**Cinnamon** 通常可直接使用
- Wayland 下的具体行为取决于合成器对 SNI 的实现

如果在 Linux 上看不到托盘图标，请优先确认以上依赖和扩展是否就位。

### Android

不支持系统托盘。
