---
title: 虚拟盘（VD）
description: 在文件管理器里像磁盘一样浏览画册与图片文件（桌面三端）。
---

「虚拟盘（VD）」会把你的图库以一个磁盘或目录的形式挂载到系统文件管理器里。你可以像浏览普通文件夹一样浏览画册和图片，适合拖拽、复制路径、被外部工具打开等场景。

VD 在 Windows、macOS、Linux 三端都可用（均为 Standard 模式）；Android 不支持，Light 版构建也不包含此功能。

## 前置依赖

不同平台依赖不同的用户态/内核态 FUSE 实现。安装方式请参见 [安装与首次启动](/guide/installation/)。

| 平台 | 依赖 | 常见安装方式 |
|------|------|-------------|
| Windows | Dokan 2.x Runtime + 内核驱动 `dokan2.sys` | 使用官方安装器安装 Dokan 2.x；仅有 `dokan2.dll` 是不够的 |
| macOS | macFUSE | `brew install macfuse` |
| Linux | fuse3 | deb 包已声明 `fuse3` 依赖；或手动 `sudo apt install fuse3` |

:::caution
若 macFUSE / fuse3 未安装，挂载会失败并提示 fuser 错误；请先安装对应驱动。
:::

Windows 下如果出现「Dokan 驱动不可用」或「Dokan 版本不兼容」的中文错误提示，说明驱动未装或版本不匹配，请安装 Dokan 2.x 最新稳定版 Runtime/Driver，安装后建议重启系统。

## 开启与关闭

前往「设置」→「应用设置」→「画册盘」。

1. 在右侧输入框填写挂载点：
   - Windows：填盘符，如 `K:`、`K:\` 或单字母 `K`（会自动规范化为 `K:\`）
   - macOS / Linux：填目录路径，默认 `~/kabegame-vd`
2. 切换左侧开关打开虚拟盘。成功时会提示「画册盘已开启」。
3. 再次切换开关即可卸载。

:::note
挂载点输入框在 VD 开启状态下是禁用的。**想修改挂载点必须先关闭 VD，改完再打开。**
:::

占位文本示例 `例如 K:\ 或 K:` 是 Windows 语境；在 macOS/Linux 上可以忽略这个占位，按目录路径填写即可。

### 自动挂载与卸载

- 应用启动时会按 `albumDriveEnabled` 设置自动挂载上次使用的挂载点；挂载失败不会阻塞 UI。
- 应用退出时会自动卸载。Windows 下若进程异常退出，盘符可能残留，需要重启或使用 `dokanctl` 清理；macOS/Linux 下一次挂载前会自动尝试 `umount` / `fusermount3 -u` 清理残留。

## 打开虚拟盘

- 在「画册列表」页右上角点击「去VD查看」
- 在「画册详情」页右上角点击「去VD查看」，会直接跳到对应画册目录
- 或在设置页「画册盘」右侧点击「打开」按钮
- 也可以直接在文件管理器地址栏输入挂载点路径

:::note
「去VD查看」按钮只在 VD 已开启、且不在 Android / Web / Light 模式下显示。
:::

## 挂载点根目录结构

进入挂载点（如 `K:\` 或 `~/kabegame-vd`），你会看到 7 个目录入口：

### 全部

包含画廊中的所有图片，按 id 升序扁平分页。

### 按插件

按收集源插件分组（如 konachan、anihonet-wallpaper 等）。每个插件一个子目录。

### 按时间

按图片收集时间分组，目录结构为 `年 / 月 / 日 / 分页`（例如 `2026年 / 4月 / 24日 / ...`）。月份与日期目录名随应用语言切换。

### 按任务

按收集任务分组，子目录名为「插件显示名 - 任务 ID」。

### 按畅游

按「畅游」浏览记录分组，按 `host` 区分。

### 按种类

下有「图片」和「视频」两个子目录，按媒体类型分开。

### 画册

包含你创建的所有画册，目录名即画册名。支持在 VD 的「画册」根下创建新目录（即新画册），也可以通过删除整个画册目录来删除画册。

:::caution
在 VD 里「删除画册目录」会被映射为真实的「删除画册」操作。请不要把它当作一般的「从视图里移除」使用。
:::

:::note
目录名会随应用语言切换而切换。如果你把 VD 路径写进外部工具的配置（例如 Wallpaper Engine），更改应用语言会让这些配置里的绝对路径失效。
:::

## 只读与受限写

VD 是只读视图：

- 图片文件本身只读，无法写入、删除、重命名、移动
- 在「画册」根下允许创建目录 = 新建画册
- 删除整个画册目录 = 删除画册
- 其他路径下的写/删/改操作都会返回拒绝

Windows 下 VD 卷容量会显示「几乎满」（总 1KB / 可用 0B），这是刻意的伪造值，并不是磁盘真的满了。

## 平台差异

| 能力 | Windows | macOS | Linux | Android |
|------|---------|-------|-------|---------|
| 虚拟盘可用 | ✅ Dokan 2.x | ✅ macFUSE | ✅ fuse3 | ❌ |
| 默认挂载点 | `K:\` | `~/kabegame-vd` | `~/kabegame-vd` | — |
| 启动时自动挂载 | ✅ | ✅ | ✅ | — |
| 即时刷新文件管理器 | ✅ | ❌ 需手动刷新 | ❌ 需手动刷新 | — |
| 挂载失败中文详情 | ✅ | ❌ 仅透出 fuser 错误 | ❌ 仅透出 fuser 错误 | — |
| `kabegame-cli vd mount` | ✅ | ❌ 目前不可用 | ❌ 目前不可用 | — |

:::note
在 macOS / Linux 上，应用层的图片变更（下载完成、删除等）**不会主动通知文件管理器刷新**。如果看不到新文件，请手动按 F5 或重新打开目录。
:::

:::note
命令行 `kabegame-cli vd {mount|unmount|status}` 子命令目前仅在 Windows 上可用。macOS / Linux 上 CLI 的 VD 子命令会返回「Virtual drive is not available」，仍需通过 GUI 设置开关挂载。
:::

## Android 不支持

Android 版本完全不编译 VD 模块，设置里也没有「画册盘」这一项。这是由 Cargo feature 与 cfg 双重屏蔽决定的，不是 bug。

## 排障

- **现象**：Windows 上挂载失败提示「Dokan 驱动不可用」或「can't install driver」。**原因**：只装了 `dokan2.dll`，未装内核驱动 `dokan2.sys`。**操作**：安装 Dokan 2.x Runtime/Driver 完整包，安装后重启系统。
- **现象**：Windows 上挂载失败提示「Dokan 版本不兼容」。**原因**：系统 Dokan 版本与应用要求不匹配。**操作**：安装最新稳定版 Dokan 2.x。
- **现象**：Windows 上挂载后迟迟没出现盘符，提示「挂载确认超时」。**原因**：Dokan 驱动可能异常或盘符冲突。**操作**：关闭开关卸载后换一个不常用的盘符（避开 C/D/E/F）再重试。
- **现象**：macOS / Linux 上挂载失败，错误文本来自 fuser（如 dlopen 失败）。**原因**：macFUSE / fuse3 未安装。**操作**：按 [安装与首次启动](/guide/installation/) 安装对应驱动。
- **现象**：macOS / Linux 上报「挂载点不是目录」或提示是残留挂载点。**原因**：上次异常退出没有卸载干净。**操作**：手动执行 `fusermount3 -u <path>`（Linux）或 `umount <path>`（macOS）清理后再重试。
- **现象**：挂载点输入框无法编辑。**原因**：VD 已开启。**操作**：先关闭 VD，再改挂载点，再重新开启。
- **现象**：看不到「去VD查看」按钮。**原因**：VD 未开启，或当前是 Android / Web / Light 模式。**操作**：到设置里开启「画册盘」；Light / Android 版本不支持此功能。
