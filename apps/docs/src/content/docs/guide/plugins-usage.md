## 什么是插件？

插件（也叫「源」）是 Kabegame 的核心功能，用于从各种网站收集图片资源。每个插件定义了如何从特定网站爬取图片，包括：目标 URL 的解析、图片链接的提取、元数据的获取等。

通过安装不同的插件，你可以从多个来源收集图片，统一管理在你的画廊和画册中。

### 插件文件格式

插件以 `.kgpg` 文件格式提供，本质上是一个 ZIP 压缩包，包含以下文件：

| 文件 | 必需 | 说明 |
|------|------|------|
| `manifest.json` | 是 | 插件元数据（名称、版本、描述、配置变量等） |
| `crawl.rhai` | 是 | 爬取逻辑脚本（Rhai 脚本语言） |
| `icon.png` | 否 | 插件图标 |
| `README.md` | 否 | 插件使用说明 |

### 插件的类型

- **网络爬取插件**：从在线网站爬取图片（如 Konachan、Anihonet 等）
- **本地导入插件**：从本地文件系统导入图片（内置）
- **自定义插件**：用户自己编写的插件，可以爬取任何支持的目标网站

### 如何获取插件？

- **从商店安装**：在「源管理」页面浏览商店源，找到喜欢的插件直接安装
- **手动导入**：如果你有 `.kgpg` 文件，可以通过拖入窗口、双击文件、或在源管理页面导入
- **自己编写**：使用插件编辑器编写自己的插件（需要了解 Rhai 脚本语言）

## 插件商店与商店源

在侧边栏进入「源管理」页面，这里集中管理「源插件」（爬虫/导入等功能的来源）。

### 已安装源 vs 商店源

- **已安装源**：你当前已经安装好的源插件列表，可在这里查看/卸载
- **商店源**：按「源名称」分 tab 展示远程可安装列表，支持安装/更新到本地

### 添加商店源

源管理页面底部有一个「添加源」的 tab，用来添加「商店源」。商店源本质上是一个可访问的 `index.json` 地址（推荐指向 GitHub Releases 资产直链）。

添加后会在顶部自动出现对应的商店 tab；进入该 tab 时会懒加载拉取该源的插件列表。

### 安装/更新插件

在某个商店源 tab 里选择插件进行安装/更新，安装成功后会同步到「已安装源」。

如果你已经导入过某个插件（`.kgpg`），商店也会显示其已安装版本信息（用于判断是否需要更新）。

### 插件编辑器

源管理页右上角有「插件编辑器」按钮，用于开发/调试插件。

## 插件导入方法

### 方法一：双击插件文件导入

在资源管理器中直接双击 `.kgpg` 插件文件，会自动打开导入窗口。这是最快捷的方式，适合从下载目录或网盘直接安装插件。

:::note
首次双击时，Windows 可能会询问「选择打开方式」，选择 Kabegame 即可。之后双击会自动关联。
:::

### 方法二：拖入主窗口

把 `.kgpg` 文件直接拖到主窗口中，应用会自动识别并导入。可以同时拖入多个插件文件，应用会批量处理。

### 方法三：从源管理页面导入

在侧边栏进入源管理页面，点击右上角的「导入源」按钮。在弹出的文件选择对话框中，选择要导入的 `.kgpg` 文件即可。

### 方法四：通过插件编辑器自行编写并导入

在侧边栏进入源管理页面，点击右上角的「插件编辑器」按钮。在插件编辑器中：

1. 创建新插件或编辑已有插件
2. 编写 `crawl.rhai` 爬虫脚本（Rhai 语言）
3. 配置 `manifest.json` 和 `config.json`
4. 添加插件图标和文档
5. 使用编辑器内置的打包功能生成 `.kgpg` 文件
6. 打包完成后，可以直接在编辑器内导入，或使用上述其他方法导入

### 导入后的处理

无论使用哪种方法，导入成功后：

- 插件会自动安装到已安装源列表中
- 如果插件已存在（同名），会提示是否覆盖/更新
- 导入完成后，可以在「源管理」的「已安装源」tab 中查看和管理

## 使用插件收集图片

### 打开收集对话框

在画廊页面，点击右上角的「收集」按钮（或空状态下的「开始导入」按钮），打开收集对话框。这是创建爬取任务的主要入口，用于配置插件参数并开始收集图片。

### 选择插件源

在收集对话框中，从「选择源」下拉菜单中挑选已安装的插件。下拉菜单会显示所有已安装的源插件。

:::note
如果下拉菜单为空，说明还没有安装任何插件。请先到插件商店安装插件。
:::

### 配置插件变量

选择插件后，如果该插件定义了配置变量，会在插件配置区域显示。常见的配置类型包括：

| 类型 | 说明 |
|------|------|
| 文本输入 | 目标 URL、搜索关键词等 |
| 数字输入 | 页码、数量限制等（支持最小值/最大值限制） |
| 下拉选择 | 从预设选项中选择 |
| 开关 | 布尔值配置 |
| 多选 | 列表或复选框类型 |
| 文件/文件夹路径 | 通过浏览按钮选择本地路径 |

根据插件的要求填写这些配置，必填项会标注红色星号（*）。

### 设置输出目录和画册

- **输出目录**：指定图片保存位置。留空则使用默认下载目录（可在设置中配置）
- **输出画册**：选择将收集的图片添加到哪个画册，或选择「+ 新建画册」创建新画册。留空则仅添加到画廊

### 保存运行配置（可选）

在运行配置下拉菜单中，可以保存当前的任务配置，方便下次快速复用。保存配置后，下次创建任务时可以直接选择该配置，无需重新填写所有参数。

:::note
如果插件版本更新导致配置不兼容，会显示「不兼容」标签，需要重新配置。
:::

### 开始收集和查看进度

配置完成后，点击「开始收集」按钮创建任务。任务创建后：

- 可以在右下角的任务抽屉中查看任务列表和进度
- 任务状态包括：等待中、运行中、已完成、失败、已取消
- 点击任务可以查看详情，包括已收集的图片和失败记录
- 运行中的任务可以随时停止

收集完成后，图片会自动保存到指定目录，并添加到画廊（和画册，如果指定了）。

### 常见问题

**Q: 任务一直显示「等待中」？**

A: 检查是否达到最大并发下载数限制（在设置中配置）。如果已有多个任务在运行，新任务会排队等待。

**Q: 任务失败怎么办？**

A: 点击任务查看详情，可以查看失败原因。常见原因包括网络错误、插件配置错误、目标 URL 无效等。可以修改配置后重试失败项。
