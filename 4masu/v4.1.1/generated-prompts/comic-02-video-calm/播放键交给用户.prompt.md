Kabegame 发布漫画世界观：

这是一个轻量异世界魔法设定，但核心仍然是现代二次元壁纸管理软件。Kabegame 应用界面像一座“壁纸魔法工坊”，左侧导航是通往不同房间的传送门，画廊是壁纸陈列大厅，画册是收藏仓库，插件是召唤新壁纸的魔法小店，任务抽屉是远征委托板，设置页是魔法调律室。

看板娘kamechan(かめちゃん)是“壁亀 / Kabegame”的桌面守护者，也像工坊里的小管理员。她不会战斗，不是魔法少女拯救世界，而是用温柔、实用、带一点小魔法的方式整理壁纸收藏。她的魔法多半表现为：路径树发光、缩略图自动排队、相册盒子展开、插件小精灵跳出来、分页器像轨道一样移动、设置滑杆调节背景雾气。

小乌龟壳背包精灵是她的搭档。它和独立行动的小乌龟助手是同一个存在的两种形态，不是两个角色。平时它背在看板娘背上，是可爱的小乌龟壳背包；需要帮忙时，它可以从背包形态化为小乌龟助手，拿路牌、盖章、推小车、举检查表、搬相册盒子、压住乱飞的刷新纸条。

同一格里只能选择一种形态：如果看板娘背上已经有小乌龟壳背包，就不要再画独立的小乌龟助手；如果小乌龟助手已经离身行动，看板娘背上就不要再画同款背包。它的存在要治愈和轻松，不要变成真正的宠物主角或战斗伙伴。

PathQL 魔女：

PathQL 是 Kabegame 工坊里的“路径魔女”，是一个可爱、聪明、稍微有点小得意的魔法角色。她负责把复杂的壁纸数据变成可以走的路径，把画廊、画册、任务详情、插件和虚拟磁盘连接到同一套路径规则里。她会施展路径咒语，让 Provider 过滤树发光、分页轨道对齐、插件入口自动接到正确房间。

PathQL 魔女不是反派，也不是战斗角色。她更像架构师、图书管理员和魔法工程师的混合体：手里拿着路径魔法书、卷轴或发光路标，身边漂浮着 `/gallery`、`/albums`、`/plugins`、`fetch`、`count`、`children` 这类短路径符号。她的魔法特效应该是细细的路径线、发光节点、目录树、书签和小型传送门，而不是巨大的攻击魔法。

PathQL 魔女和 kamechan 的关系是“帮忙整理工坊的前辈/伙伴”。看板娘负责把壁纸收藏变得亲切可爱，PathQL 魔女负责让背后的路径和规则稳定运转。两人同框时，看板娘更元气治愈，PathQL 魔女更理性、俏皮、像在说“路径已经安排好了”。

异世界魔法元素要轻量：

- Provider 过滤树可以像发光的魔法路径树。
- PathQL / DSL 可以像工坊里的路径咒文或目录魔法书。
- Provider 路径则像咒语。
- 插件是一个个呆萌可爱的小精灵，不同网站的插件有不同的配色。
  - 插件可以拟人化成小小的“插件小精灵”或“魔法工具包”，从插件小店、召唤门、拼图卡片里探出头。它们负责带回不同站点的壁纸线索，例如 Pixiv、Konachan、anime-pictures 等，但只作为 UI 卡片和功能的可爱化表现。
  - 插件小精灵的动作可以是举着站点路牌、递出参数表单、背着小网兜收集缩略图、把自己的 provider 路径挂到画廊树上。它们应该像工坊助手，不要画成战斗召唤兽或独立主角。
  - 不同插件小精灵可以有轻微差异，例如颜色、徽章、工具包图标不同，但都要保持简洁可读，避免过多角色堆满画面。插件页仍然要看得出是插件卡片网格和安装/配置入口。
  - Pixiv 小精灵：像认真整理排行榜的插画邮差，带蓝白色 Pixiv 徽章、小画夹和标签卡。它擅长“排行榜、收藏、画师作品、关键词搜索”，可以抱着日榜卷轴、画师名片、书签盒或关键词放大镜登场。涉及登录、Cookie、UID、R18 等高级设置时，用“门票、通行证、年龄认证印章”等轻量 UI 隐喻表达，不要画成危险或擦边内容。
  - 米游社 / HoYoLAB 小精灵：像跨游戏分区的同人图向导，带星形或游戏分区徽章，手里拿“原神、崩坏：星穹铁道、绝区零”等游戏分区路牌和关键词搜索卷轴。它擅长按游戏分区与关键词寻找同人图、壁纸，也能搬来帖子 URL、作者 UID、评论区图片小盒子。画面中可以把它表现为在多个游戏门牌之间穿梭的向导。
  - bilibili 专栏小精灵：像戴小电视徽章的专栏剪报员，背着文章卷轴、cv 链接卡、图文帖卡片和评论气泡。它擅长关键词搜索专栏、读取单篇 cv/链接、从正文里提取图片，并把评论作为详情展示素材。它可以举“综合排序、最新发布、最多点击、最多喜欢、最多评论”等排序牌，但不要把画面画成视频播放器主界面。
  - 如果插件小精灵在四格漫画中登场，剧情描述必须明确写出登场人物、站位、动作和它在做什么。例如“Pixiv 小精灵抱着日榜卷轴从插件卡片里跳出来，把缩略图交给看板娘”，不要只写对白。对白只是辅助，不能替代画面描述。
- 画册可以像收纳宝箱、相册房间或轮播舞台。
- 任务可以像远征委托，进度条像魔法能量槽。
- 应用内背景图可以像把壁纸铺成舞台幕布。
- 设置项可以像调律仪表、滑杆和开关咒具。

但不要让画面变成传统奇幻冒险。不要出现战斗、怪物、王国战争、魔王、法杖大招、复杂魔法阵。魔法元素只是帮助解释软件功能的可爱隐喻，主角仍然是kamechan、PathQL魔女等角色。不要画其他没有提到的插件。

画面比例建议：现代软件 UI 占主体，魔法元素作为装饰和视觉隐喻。UI 结构必须清楚，不能被魔法特效遮住。整体氛围是“二次元开源壁纸工坊的一天”，轻松、治愈、带一点小コント。

用途：

当漫画里需要表现“这是 Kabegame 应用本体”时使用。适合任何页面的背景，也适合让看板娘站在左侧导航栏前做导游。

整体布局：

桌面端是现代应用窗口。左侧是固定导航栏，右侧是主内容区。应用窗口可以有轻微圆角和浅色背景。主内容区可以叠加柔和的应用内壁纸背景，背景经过透明度和模糊处理，不喧宾夺主。

左侧导航细节：

左侧导航顶部显示 Kabegame 小图标和“Kabegame”标题。导航项使用图标 + 短文字：画廊、画册、插件、畅游、运行配置、设置、帮助。导航栏可以折叠，折叠状态下只显示图标。当前页面的导航项有浅紫色高亮。

主内容区细节：

右侧主内容区占据大部分空间，页面顶部通常有标题或工具栏，下面是主要内容。内容可能是画廊缩略图网格、画册卡片网格、插件卡片网格、设置表单、任务详情等。视觉上不要像浏览器网页，也不要像传统文件管理器，要像专门为二次元壁纸收藏设计的应用。画廊的图片为纯图，并没有什么额外的指示标志。

全局浮层：

可以出现半透明抽屉、弹窗或悬浮层，例如快速设置抽屉、帮助抽屉、任务抽屉、导入确认弹窗。它们从右侧或中间浮出，使用 Element Plus 风格的按钮、开关、标签页和卡片。

看板娘互动建议：

看板娘可以站在左侧导航旁拿导游旗；可以从导航栏把用户带到画廊；也可以坐在主内容区边缘整理缩略图。小乌龟壳背包精灵如果化为小乌龟助手，可以趴在左侧导航顶部，像应用守护者；如果它仍是背包形态，就不要再画独立小乌龟。

避免：

不要把主框架画成网站 landing page。不要出现大面积营销 hero。不要让左侧导航消失。不要把导航项写成一堆小到不可读的文字。不要把主内容区画成纯空白。

## 页面布局设定

Kabegame 画廊页布局 prompt：

用途：

当漫画主题与图片收集、图片筛选、图片尺寸 / 格式 / 名称、壁纸设置、视频缩略图、空状态相关时使用。画廊是 Kabegame 的核心页面。

页面结构：

左侧保留 Kabegame 主导航。右侧主内容区是画廊，并分成三层：第一层是画廊标题和主要操作行，第二层是浏览控制行，第三层是图片网格。不要把所有按钮塞进同一条工具栏。

第一层标题行左侧显示“画廊”和图片总数，例如“共 2345 张”。同一行右侧放主要操作按钮：刷新、收集、导入、快速设置、帮助。这一行像页面 header，负责说明当前页面和提供高频全局操作。

第二层浏览控制行放筛选、排序、分页、每页数量和搜索框。布局顺序应接近：筛选、排序、上一页 / 当前页 / 下一页、每页 100、搜索图片。它是画廊内容的控制条，视觉上比标题行更低一级。

第三层是大量二次元壁纸缩略图组成的多列网格。图片卡片排列整齐，像壁纸收藏库。

顶部工具栏具体元素：

标题行的“共 2345 张”靠近页面标题，表示当前画廊总图片数。刷新按钮用圆形箭头图标；收集按钮表示网络爬取；导入按钮表示本地图片 / 视频 / 文件夹导入；快速设置按钮用齿轮图标；帮助按钮用问号图标。这些按钮可以是小型 Element Plus 风格按钮，不要画得很大。

浏览控制行具体元素：

筛选按钮（展开provider过滤树）可以带漏斗图标，排序按钮可以带上下箭头。分页器紧跟在筛选和排序后面，显示“‹ 1 / 24 ›”，左右箭头用于翻页。每页数量显示为“每页 100”或下拉选择。搜索框放在这一行偏右位置，提示文字是“搜索图片...”，用于按图片名称查找。

分页器：

分页器属于第二层浏览控制行，不要单独画成很大的底部分页条。它应该轻量、紧凑，形式接近“‹ 1 / 24 ›”，旁边配合“每页 100”和搜索框。图片总数已经在标题行显示，不要在分页器里重复强调总数。

空状态：

画廊为空时，主内容区中央显示看板娘拿着空白图片卡，旁边是空相册盒子，按钮写“开始收集”或留白。台词可为“啊哦，这里还什么都没有呢”。

看板娘互动建议：

看板娘可以抱着一摞缩略图，把它们放进网格；也可以拿放大镜站在搜索框旁；可以把“PNG / JPG / 1920x1080”标签贴到图片角落；可以趴在预览大图边框旁检查细节；小乌龟壳背包精灵如果化为小乌龟助手，可以推着分页器、举着播放按钮或递出右键菜单小牌子。

避免：

不要把画廊画成普通照片墙而没有工具栏。不要只画一个大图而没有网格。不要在预览层底部画一排具体功能按钮。不要让图片缩略图过于杂乱导致页面结构看不出来。不要出现过多不可读小字。

# 播放键交给用户
漫画主题：视频缩略图安静待机

候选类型：无笑点说明型短故事

为什么适合画：视频自动播放优化很适合画成“缩略图会议安静下来”的视觉梗；尺寸回填则可以用量尺和比例框表现，普通用户也能理解。

剧情角度：无笑点/说明型短故事。展示视频缩略图从自动播放改为安静待机，并通过尺寸回填让网格比例更稳定。

对应更新点：
- 视频图片项不再自动播放，ImageItem 中视频缩略图改为右上角播放/暂停徽章，由用户手动切换播放。
- v012 迁移会为已有 mp4/mov 行回填 width 和 height，缺失、损坏或 Android content:// 文件会跳过而不阻断迁移。
- 图片详情和网格可以利用 width/height 显示或计算更正确的尺寸与比例，减少视频缩略图排版不准。


整体目标：

生成日系二次元应用宣传四格漫画的单格画面。最终四格会被组装成 2x2 漫画，阅读顺序是左上、右上、左下、右下。每次只生成其中一格，画面要像画师为开源壁纸管理工具画的轻量更新公告，不要像 AI 自动堆满细节的插画。重点是角色表情、清楚的 UI 结构、短小的剧情推进、舒服的留白和可读的对白。

最终四格版式：

- 最终成品是 2x2 四格漫画，不是竖向长图。
- 阅读顺序固定为：第 1 格左上，第 2 格右上，第 3 格左下，第 4 格右下。
- 每格固定为横向 3:2，推荐生成尺寸为 1536x1024 px。
- 四张单格拼成最终 2x2 成品时，推荐总尺寸为 3072x2048 px，仍然保持 3:2。
- 四格尺寸必须一致，不要混用不同分辨率或不同裁切比例。
- 每格边框清楚，圆角轻微，不要厚重黑框。
- 每格只讲一个动作或一个小变化，不要在同一格塞多个时间点。
- 每格都要留出对白框位置，避免角色、UI、对白互相遮挡。

单格生成模式，默认使用：

- 每次只生成一张单格画面，不要一次生成 2x2 四格。
- 单格画面本身使用 1536x1024 px，保持 3:2 横向比例，方便后期拼成 2x2。
- 单格可以有自己的圆角边框，但不要画出其他三格的边框、草图或占位格。
- 提示词必须明确写出当前要生成的是第几格，以及它在最终 2x2 中的位置。
- 第 1 格，左上：开场。交代 Kabegame 应用场景、主角站位、这次更新主题的起因或小问题。不要提前画出结论。
- 第 2 格，右上：展开。功能开始工作，UI 结构或视觉隐喻变清楚。
- 第 3 格，左下：转折。加入轻微小コント或误会，但笑点要服务功能。
- 第 4 格，右下：收束。应用整理完成，界面清爽，角色轻松吐槽或微笑。
- 每一格单独生成时，都要保持同一套角色设计、色调、线稿、边框和 UI 简化程度，让四张图拼在一起仍然像同一篇漫画。

主角固定：

Kabegame 看板娘 kamechan。她是“壁亀 / Kabegame”的桌面守护者，也是壁纸管理工坊的小管理员。

角色外观：

- 浅紫色短发，发尾柔软，整体发型清爽。
- 紫粉色眼睛，表情活泼、治愈、稍微带一点小得意。
- 水手服风格上衣，可以有轻量开源宅系小配饰。
- 头发上有小乌龟造型发饰或小乌龟头发符号，必须可爱但不要过大。
- 身形偏 chibi / moe，但不要过度幼化。四格整体中至少有一次能看见腿部和完整站姿，不必每格都全身。
- 动作像应用里的小管理员：拿导游旗、贴标签、整理缩略图、点击筛选、抱相册盒、给插件盖章、看任务进度。

角色气质：

可爱、轻快、亲切、有一点软件吉祥物的营业感。她不是战斗魔法少女，不是偶像舞台主角，不是单纯站在空背景里的看板立绘。她应该和 Kabegame 应用界面发生互动。

小乌龟壳背包精灵：

小乌龟壳背包和乌龟助手是同一个精灵的两种形态，不是两个角色。平时它背在 kamechan 背上，是一个可爱的小乌龟壳背包；需要帮忙时，它可以从背包形态化为小乌龟助手，拿牌子、搬小盒子、推小车、盖章、举检查表、收拾多余纸条。

同一格里只能选择一种形态：如果 kamechan 背上已经有小乌龟壳背包，就不要再画独立的小乌龟助手；如果小乌龟助手已经离身行动，kamechan 背上就不要再画同款背包。它是辅助笑点，不要抢主角，不要变成宠物主角。

基础视觉风格：

- 高质量日系 anime mascot comic。
- 干净线稿，线条有轻微手绘感，不要像矢量贴纸一样完全机械。
- 柔和 cel shading，阴影块面简单，最多两层阴影。
- 配色以粉紫、浅紫、淡奶油色为主，粉色和紫色都要出现。
- 因为主角偏紫，背景可以多放一点粉色，让画面更轻。
- 少量绿色作为乌龟主题点缀，不要把绿色变成主色。
- 整体清爽、明亮、轻松搞笑、治愈。
- 像软件更新公告小漫画，不像大型商业海报、游戏卡面、插画集封面。

绘画感控制：

为了减少 AI 味，画面要有真实绘画的取舍：

- 每格只保留 1 个主动作、1 个主要 UI 区域、1 到 3 个辅助小物。
- 背景允许有大块干净色面，不需要每个角落都有图标、粒子、花纹或光效。
- UI 图标只画关键图标，例如画廊、画册、插件、路径树、下载、收藏、缩略图、分页器。
- 缩略图网格可以概括成整齐小色块，不要每张缩略图都画成完整插画。
- 魔法特效只用细线、发光节点、小星点，数量少而清楚。
- 线稿可以有轻微粗细变化和少量不完全对称，保持自然手绘感。
- 不要过度锐化，不要超高密度装饰，不要满屏发光，不要把背景画成复杂机械城市。
- 不要让衣服、头发、UI、背景都同时有复杂渐变和复杂纹样。
- 画面应该像“画师画得很干净”，不是“模型不知道停在哪里”。

构图原则：

- 主角通常占每格高度的 45% 到 70%，不要被 UI 淹没。
- UI 是背景和剧情道具，不是完整截图。结构要清楚，但细节要简化。
- 应用窗口可以占画面后方或一侧，保留左侧导航、主内容区、工具栏或卡片网格的基本轮廓。
- 每格视角轻微变化即可，不要频繁大透视、鱼眼、夸张镜头。
- 至少有一格让角色和 UI 产生明确互动，例如指向筛选树、把缩略图放进相册、给插件卡盖章。
- 白框或气泡周围留出干净空间，中文对白要容易后期添加。

对白和文字：

- 中文对白放在圆形或椭圆形对白框中。
- 字数要短，像四格漫画台词，不要长说明。
- 如果模型不擅长中文，可以让对白框留白，后期再加字。
- UI 标签只保留短词，例如“画廊”“画册”“插件”“设置”“筛选”“尺寸”“格式”“名称”“路径 OK”。
- 不要出现乱码、伪中文、密密麻麻的小字、长代码块、长 URL。

应用 UI 元素：

可以出现可爱的简化软件元素：

- 壁纸缩略图网格。
- 画册卡片、封面拼贴、小相册盒。
- 插件卡片、拼图图标、小工具包。
- Provider 路径树、筛选树、分页器、搜索框。
- 下载进度、任务进度条、成功/失败小徽章。
- 收藏星标、导入按钮、设置滑杆、开关。

这些元素要像漫画道具，而不是完整真实截图。每格最多选择其中一到两类作为主视觉。

轻量魔法世界观：

Kabegame 应用可以像一座“壁纸魔法工坊”。魔法只是软件功能的可爱隐喻：

- 筛选树像发光路径树。
- 画册像收纳盒或小房间。
- 插件像小工具包或插件小精灵。
- 任务像远征委托板。
- 设置像调律仪表。

不要变成传统奇幻冒险。不要战斗、怪物、王国战争、巨大魔法阵、武器、攻击技能。

四格节奏：

完整四格漫画建议这样推进：

第 1 格，左上：开场。展示 Kabegame 应用页面和这次主题的小问题或新功能入口。看板娘发现问题、拿出道具或带用户进入页面。

第 2 格，右上：展开。功能开始工作，UI 结构变清楚，路径树、相册、插件、任务或设置项以可爱的方式出现。

第 3 格，左下：转折。加入一个轻微小コント，例如背包精灵化为小乌龟助手后拿错牌、缩略图排队太认真、插件小工具包急着上场。笑点要服务功能，不要偏题。

第 4 格，右下：收束。应用整理完成，界面清爽，角色轻松吐槽或微笑。最后一格要看出更新带来的好处。

避免项：

- 单格生成时，不要一次生成 2x2 版式。
- 不要生成竖向四格长图。
- 不要把画面画成网站 landing page 或营销 hero。
- 不要把 Kabegame 画成聊天软件、浏览器首页、普通文件管理器或游戏战斗界面。
- 不要把看板娘画成性感暴露角色。
- 不要让 PathQL、Provider、DSL、metadata 变成满屏代码墙。
- 不要塞满随机按钮、随机图标、随机科技面板。
- 不要出现无意义英文水印、伪 logo、大段说明文字。
- 不要每个角落都放星星、纸片、彩带、漂浮窗口。
- 不要把阴影和高光做成厚重 3D 渲染。
- 不要画成过度精修的 AI 插画封面，保持轻松的漫画公告感。

最终输出要求：

1536x1024 px 单格横向漫画画面，最终可拼成 3072x2048 px 的 2x2 四格宣传漫画。柔和粉紫色调，清楚圆角边框，干净线稿，轻量手绘感，应用 UI 简化可读，角色表情丰富，信息量适中，有留白，有短对白框。整体像 Kabegame 插件 provider trees 漫画的同系列作品。
