日系二次元 4 格竖排漫画，纵向长图，比例约 2:3 ，四个横向分镜从上到下排列，每格高度接近一致，分镜边框清晰圆角，整体像软件更新公告宣传漫画。

角色固定：

Kabegame 看板娘，浅紫色短发，紫粉色眼睛，活泼治愈的笑容，紫红色兜帽外套，内搭水手服风格上衣，背着小乌龟壳背包，头发上有小乌龟发饰，兜帽侧面也有乌龟图案。整体气质可爱、轻快、开源宅系、二次元壁纸管理工具吉祥物。她是“壁亀 / Kabegame”的桌面守护者。

风格固定：

高质量日系 anime mascot comic，干净线稿，柔和粉紫配色，淡奶油色背景，少量绿色作为乌龟主题点缀。画面轻松搞笑、治愈、软件吉祥物宣传漫画风格。细节精致但不厚重，UI 元素可爱，像壁纸仓库、相册、插件、路径树、下载、收藏、缩略图等软件元素以可爱图标形式出现。

构图固定：

四格漫画竖排，每一格都有明确情节推进。角色表情丰富，偏 chibi / moe 但不要过度幼化。画面信息量适中，不能太乱。每格保留足够留白放中文台词。中文台词放在圆形或椭圆形对白框中，字体清晰可读，不要出现乱码，不要出现多余大段文字。

整体尺寸保持为竖向四格宣传漫画，角色与分镜风格参考上一张 Kabegame 插件 provider trees 漫画，保持同样的柔和粉紫色调、圆角边框、可爱软件 UI 元素和轻松搞笑节奏。

Kabegame 发布漫画世界观：

这是一个轻量异世界魔法设定，但核心仍然是现代二次元壁纸管理软件。Kabegame 应用界面像一座“壁纸魔法工坊”，左侧导航是通往不同房间的传送门，画廊是壁纸陈列大厅，画册是收藏仓库，插件是召唤新壁纸的魔法小店，任务抽屉是远征委托板，设置页是魔法调律室。

看板娘是“壁亀 / Kabegame”的桌面守护者，也像工坊里的小管理员。她不会战斗，不是魔法少女拯救世界，而是用温柔、实用、带一点小魔法的方式整理壁纸收藏。她的魔法多半表现为：路径树发光、缩略图自动排队、相册盒子展开、插件小工具跳出来、分页器像轨道一样移动、设置滑杆调节背景雾气。

小乌龟壳背包是她的搭档，像一个认真但可爱的工坊助手。它可以拿路牌、盖章、推小车、举检查表、搬相册盒子、压住乱飞的刷新纸条。它的存在要治愈和轻松，不要变成真正的宠物主角或战斗伙伴。

PathQL 魔女：

PathQL 是 Kabegame 工坊里的“路径魔女”，是一个可爱、聪明、稍微有点小得意的魔法角色。她负责把复杂的壁纸数据变成可以走的路径，把画廊、画册、任务详情、插件和虚拟磁盘连接到同一套路径规则里。她会施展路径咒语，让 Provider 过滤树发光、分页轨道对齐、插件入口自动接到正确房间。

PathQL 魔女不是反派，也不是战斗角色。她更像架构师、图书管理员和魔法工程师的混合体：手里拿着路径魔法书、卷轴或发光路标，身边漂浮着 `/gallery`、`/albums`、`/plugins`、`fetch`、`count`、`children` 这类短路径符号。她的魔法特效应该是细细的路径线、发光节点、目录树、书签和小型传送门，而不是巨大的攻击魔法。

PathQL 魔女和 Kabegame 看板娘的关系是“帮忙整理工坊的前辈/伙伴”。看板娘负责把壁纸收藏变得亲切可爱，PathQL 魔女负责让背后的路径和规则稳定运转。两人同框时，看板娘更元气治愈，PathQL 魔女更理性、俏皮、像在说“路径已经安排好了”。

异世界魔法元素要轻量：

- Provider 过滤树可以像发光的魔法路径树。
- PathQL / DSL 可以像工坊里的路径咒文或目录魔法书。
- Provider 路径则像咒语。
- 插件是一个个呆萌可爱的小精灵，不同网站的插件有不同的配色。
  - 插件可以拟人化成小小的“插件小精灵”或“魔法工具包”，从插件小店、召唤门、拼图卡片里探出头。它们负责带回不同站点的壁纸线索，例如 Pixiv、Konachan、anime-pictures 等，但只作为 UI 卡片和功能的可爱化表现。
  - 插件小精灵的动作可以是举着站点路牌、递出参数表单、背着小网兜收集缩略图、把自己的 provider 路径挂到画廊树上。它们应该像工坊助手，不要画成战斗召唤兽或独立主角。
  - 不同插件小精灵可以有轻微差异，例如颜色、徽章、工具包图标不同，但都要保持简洁可读，避免过多角色堆满画面。插件页仍然要看得出是插件卡片网格和安装/配置入口。
  - Pixiv 小精灵：像认真整理排行榜的插画邮差，带蓝白色 Pixiv 徽章、小画夹和标签卡。它擅长“排行榜、收藏、画师作品、关键词搜索”，可以抱着日榜卷轴、画师名片、书签盒或关键词放大镜登场。涉及登录、Cookie、UID、R18 等高级设置时，用“门票、通行证、年龄认证印章”等轻量 UI 隐喻表达，不要画成危险或擦边内容。
  - 米游社 / HoYoLAB 小精灵：像跨游戏分区的同人图向导，带星形或游戏分区徽章，手里拿“原神、崩坏：星穹铁道、绝区零”等游戏分区路牌和关键词搜索卷轴。它擅长按游戏分区与关键词寻找同人图、壁纸，也能搬来帖子 URL、作者 UID、评论区图片小盒子。画面中可以把它表现为在多个游戏门牌之间穿梭的向导。
  - bilibili 专栏小精灵：像戴小电视徽章的专栏剪报员，背着文章卷轴、cv 链接卡、图文帖卡片和评论气泡。它擅长关键词搜索专栏、读取单篇 cv/链接、从正文里提取图片，并把评论作为详情展示素材。它可以举“综合排序、最新发布、最多点击、最多喜欢、最多评论”等排序牌，但不要把画面画成视频播放器主界面。
  - 如果插件小精灵在四格漫画中登场，剧情描述必须明确写出登场人物、站位、动作和它在做什么。例如“Pixiv 小精灵抱着日榜卷轴从插件卡片里跳出来，把缩略图交给看板娘”，不要只写对白。对白只是辅助，不能替代画面描述。
- 画册可以像收纳宝箱、相册房间或轮播舞台。
- 任务可以像远征委托，进度条像魔法能量槽。
- 应用内背景图可以像把壁纸铺成舞台幕布。
- 设置项可以像调律仪表、滑杆和开关咒具。

但不要让画面变成传统奇幻冒险。不要出现战斗、怪物、王国战争、魔王、法杖大招、复杂魔法阵。魔法元素只是帮助解释软件功能的可爱隐喻，主角仍然是 Kabegame 应用、看板娘、壁纸收藏和发布更新。

画面比例建议：现代软件 UI 占主体，魔法元素作为装饰和视觉隐喻。UI 结构必须清楚，不能被魔法特效遮住。整体氛围是“二次元开源壁纸工坊的一天”，轻松、治愈、带一点小コント。

用途：

当漫画主题与图片收集、图片筛选、图片尺寸 / 格式 / 名称、壁纸设置、视频缩略图、空状态相关时使用。画廊是 Kabegame 的核心页面。

页面结构：

第一层标题行左侧显示“画廊”和图片总数，例如“共 2345 张”。同一行右侧放主要操作按钮：刷新、收集、导入、快速设置、帮助。这一行像页面 header，负责说明当前页面和提供高频全局操作。

顶部工具栏具体元素：

标题行的“共 2345 张”靠近页面标题，表示当前画廊总图片数。刷新按钮用圆形箭头图标；收集按钮表示网络爬取；导入按钮表示本地图片 / 视频 / 文件夹导入；快速设置按钮用齿轮图标；帮助按钮用问号图标。这些按钮可以是小型 Element Plus 风格按钮，不要画得很大。

筛选按钮（展开provider过滤树）可以带漏斗图标，排序按钮可以带上下箭头。分页器紧跟在筛选和排序后面，显示“‹ 1 / 24 ›”，左右箭头用于翻页。每页数量显示为“每页 100”或下拉选择。搜索框放在这一行偏右位置，提示文字是“搜索图片...”，用于按图片名称查找。

Provider 过滤树功能与故事：

筛选按钮展开的是 Kabegame 的 Provider 过滤树。它不是普通的单选筛选菜单，而是一棵“壁纸仓库路径树”：用户可以沿着不同路径浏览同一批壁纸，例如全部图片、按日期、按插件来源、按图片格式、按图片尺寸、按图片名称、按壁纸顺序。Provider 像应用背后的向导，把复杂数据整理成可点击的目录，让画廊、画册、任务详情和虚拟磁盘都能用相似的路径方式查看图片。

漫画里可以把 Provider 过滤树画成从“筛选”按钮长出来的发光小树或目录地图。树枝上挂着短标签：“全部”“日期”“插件”“格式”“尺寸”“名称”。看板娘像图书管理员一样顺着树枝给壁纸分类，小乌龟助手负责举路牌或盖章。故事感是“图片太多也不怕，Provider 会带路”。

Provider 过滤树视觉约束：

树要像轻量弹出层或可爱的目录地图，不能画成系统文件资源管理器。节点文字尽量短，最多保留“全部、日期、插件、格式、尺寸、名称”等关键词。如果图片模型容易乱写字，可以只画图标和空标签，后期再补文字。

图片网格具体元素：

网格卡片显示二次元壁纸缩略图。卡片角落可以有小标签：PNG、JPG、MP4、1920x1080、4K、竖屏、横屏。视频缩略图中央显示播放按钮，不要自动播放得很混乱。选中图片时卡片边框变成紫色，并出现小勾选标记。

图片预览结构：

双击图片后进入预览。预览是覆盖在 Kabegame 应用上的大图层，中央显示当前图片，左右有上一张 / 下一张箭头，左上或右上有关闭按钮，右上可以有“详情 i”入口。预览层不需要画出一排具体功能按钮，因为设置壁纸、加入画册、隐藏、删除等动作主要通过图片右键菜单或选择操作完成。预览画面要保留一点应用上下文，不能像脱离应用的纯图片查看器。

右键菜单与多选操作：

右键菜单是贴近图片卡片的小型浮层，菜单项可以用短标签表现：预览、收藏、加入画册、设置壁纸、隐藏、删除。漫画里只要画出菜单形状和几个短标签，不要画成复杂工具栏。

多选图片时，可以在画廊上方或网格附近出现选择操作条，显示“已选择 N 张”，并提供收藏、加入画册、设置壁纸、隐藏、删除等动作。也可以用更漫画化的方式表现：看板娘把多张图片用丝带捆起来，小乌龟助手举着“已选择”牌子。

分页器：

分页器属于第二层浏览控制行，不要单独画成很大的底部分页条。它应该轻量、紧凑，形式接近“‹ 1 / 24 ›”，旁边配合“每页 100”和搜索框。图片总数已经在标题行显示，不要在分页器里重复强调总数。

空状态：

画廊为空时，主内容区中央显示看板娘拿着空白图片卡，旁边是空相册盒子，按钮写“开始收集”或留白。台词可为“啊哦，这里还什么都没有呢”。

看板娘互动建议：

看板娘可以抱着一摞缩略图，把它们放进网格；也可以拿放大镜站在搜索框旁；可以把“PNG / JPG / 1920x1080”标签贴到图片角落；可以趴在预览大图边框旁检查细节；小乌龟助手可以推着分页器、举着播放按钮或递出右键菜单小牌子。

避免：

不要把画廊画成普通照片墙而没有工具栏。不要只画一个大图而没有网格。不要在预览层底部画一排具体功能按钮。不要让图片缩略图过于杂乱导致页面结构看不出来。不要出现过多不可读小字。

# 插件递上自己的小地图
漫画主题：插件自己带路的路径树

候选类型：无笑点说明型短故事

为什么适合画：这是本次发布的核心架构变化，但可以包装成“插件小精灵自带地图”：用户看到的是插件页、画廊筛选树和虚拟磁盘入口变得更一致，开发者含义用后台路径树隐喻表达。

剧情角度：无笑点/说明型短故事。用插件小精灵把自己的路径地图交给 Kabegame，说明插件自带 Provider 子树、画廊按插件浏览和虚拟磁盘共享同一规则。

对应更新点：
- 安装的爬虫插件可以随包携带 provider DSL 文件，并通过自己的插件入口 provider 暴露嵌套路径子树。
- 画廊“按插件”过滤树与虚拟磁盘插件文件夹共享同一套 plugin-owned provider contract，插件特定浏览路径来自插件元数据而不是写死在 UI 里。
- PathQL provider DSL 迁移让 gallery、album、task、plugin、media-type、date、search、wallpaper-order、raw-image、image-metadata 等路径从手写路由转为声明式路径树。
- 实际组件会按需 list_provider_children 懒加载展开的插件分支，搜索或刷新时保留根节点，不把整棵树塌成单一 Loading 行。

```text
Kabegame 是一个二次元壁纸爬取、管理、设置和轮播客户端。本漫画主题是 Kabegame v4.1.0 的“插件自己带路的路径树”：安装插件可以携带自己的 provider 路径定义，并通过插件入口暴露嵌套子树；画廊按插件浏览和虚拟磁盘插件文件夹使用同一套规则。PathQL DSL 是后台整理仓库的视觉隐喻，不要把它画成真实用户需要编辑的界面。
第 1 格：插件页。看板娘站在插件卡片网格前，Pixiv 小精灵和 Konachan 小精灵从卡片里探出头，各自递出一张小地图卷轴，卷轴上只保留短标签“排行”“标签”“作者”“搜索”。小乌龟助手拿着安装印章。
第 2 格：画廊页。看板娘点击“筛选”，弹出按插件分支的路径树，插件小精灵把自己的小地图挂到“插件”树枝下面。树根保持清楚，有“全部、日期、格式、尺寸、插件”等短节点；展开的插件分支像懒加载亮起的小路。
第 3 格：同一棵路径树延伸到虚拟磁盘入口的视觉隐喻。画面一侧是画廊筛选树，另一侧是一个桌面文件夹或磁盘入口，二者之间由同样的紫色路径线连接。PathQL 魔女可在后台像图书管理员一样整理短路径牌，不要抢主角。
第 4 格：插件分支稳定展开。看板娘沿着插件自带的小路把壁纸缩略图带进网格，小乌龟助手把“Loading”小牌子收进工具箱，根节点仍然站在原位。最后一格温柔说明“以后每个插件可以带自己的路标”。
版本更新点要画出来：插件包自带路径树、画廊按插件分支、虚拟磁盘共享插件路径合同、声明式后台整理、按需展开且根节点不消失。对白建议短句：第 1 格“这次插件自己带地图来。”第 2 格“挂到按插件的树枝上。”第 3 格“画廊和磁盘走同一条路。”第 4 格“根节点还在，分支慢慢亮起来。”避免项：不要画成代码编辑器主界面；不要让 PathQL DSL 成为用户正在手写的功能；不要把插件小精灵画成战斗召唤兽；不要把所有节点写成长路径。
```
可选对白：

- 第 2 格：挂到按插件的树枝上。
- 第 3 格：画廊和磁盘走同一条路。
- 第 4 格：根节点还在，分支慢慢亮起来。
