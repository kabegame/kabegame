Kabegame 发布漫画世界观：

这是一个轻量异世界魔法设定，但核心仍然是现代二次元壁纸管理软件。Kabegame 应用界面像一座“壁纸魔法工坊”，左侧导航是通往不同房间的传送门，画廊是壁纸陈列大厅，画册是收藏仓库，插件是召唤新壁纸的魔法小店，任务抽屉是远征委托板，设置页是魔法调律室。

PathQL 魔女：

PathQL 是 Kabegame 工坊里的“路径魔女”，是一个可爱、聪明、稍微有点小得意的魔法角色。她负责把复杂的壁纸数据变成可以走的路径，把画廊、画册、任务详情、插件和虚拟磁盘连接到同一套路径规则里。她会施展路径咒语，让 Provider 过滤树发光、分页轨道对齐、插件入口自动接到正确房间。

PathQL 魔女不是反派，也不是战斗角色。她更像架构师、图书管理员和魔法工程师的混合体：手里拿着路径魔法书、卷轴或发光路标，身边漂浮着 `/gallery`、`/albums`、`/plugins`、`fetch`、`count`、`children` 这类短路径符号。她的魔法特效应该是细细的路径线、发光节点、目录树、书签和小型传送门，而不是巨大的攻击魔法。

PathQL 魔女和 Kabegame 看板娘的关系是“帮忙整理工坊的前辈/伙伴”。看板娘负责把壁纸收藏变得亲切可爱，PathQL 魔女负责让背后的路径和规则稳定运转。两人同框时，看板娘更元气治愈，PathQL 魔女更理性、俏皮、像在说“路径已经安排好了”。

  - bilibili 专栏小精灵：像戴小电视徽章的专栏剪报员，背着文章卷轴、cv 链接卡、图文帖卡片和评论气泡。它擅长关键词搜索专栏、读取单篇 cv/链接、从正文里提取图片，并把评论作为详情展示素材。它可以举“综合排序、最新发布、最多点击、最多喜欢、最多评论”等排序牌，但不要把画面画成视频播放器主界面。
  - 如果插件小精灵在四格漫画中登场，剧情描述必须明确写出登场人物、站位、动作和它在做什么。例如“Pixiv 小精灵抱着日榜卷轴从插件卡片里跳出来，把缩略图交给看板娘”，不要只写对白。对白只是辅助，不能替代画面描述。
- 画册可以像收纳宝箱、相册房间或轮播舞台。
- 任务可以像远征委托，进度条像魔法能量槽。

但不要让画面变成传统奇幻冒险。不要出现战斗、怪物、王国战争、魔王、法杖大招、复杂魔法阵。魔法元素只是帮助解释软件功能的可爱隐喻，主角仍然是 Kabegame 应用、看板娘、壁纸收藏和发布更新。

画面比例建议：现代软件 UI 占主体，魔法元素作为装饰和视觉隐喻。UI 结构必须清楚，不能被魔法特效遮住。整体氛围是“二次元开源壁纸工坊的一天”，轻松、治愈、带一点小コント。

Kabegame 应用界面公共设定：

Kabegame 是一个基于 Tauri 的二次元壁纸爬取、管理、设置和轮播客户端。它可以通过插件从 Pixiv、Konachan、anime-pictures 等二次元壁纸站点收集图片，也可以导入本地图片、视频、文件夹、压缩包或 .kgpg 插件。用户在应用里整理画廊、创建画册、预览大图、设置桌面壁纸、开启自动轮播，还可以管理爬虫任务和插件。

应用气质：

“二次元人自己的开源壁纸工具”。温柔、轻快、实用，像一只安静的小龟守护桌面壁纸收藏。画面应表现为现代桌面壁纸管理器，不是聊天软件、浏览器首页、普通文件管理器或游戏战斗界面。

角色与界面的关系：

Kabegame 看板娘不是站在空背景里，她应该像应用里的小管理员：在画廊缩略图之间整理图片，在画册卡片旁贴标签，在插件页面给插件分配小房间，在左侧导航前带路，在任务抽屉旁查看进度。小乌龟壳背包可以作为助手，拿着路径树、筛选器、扳手、放大镜、播放按钮或相册标签。

推荐组合方式：

1. 先使用 `bo.prompt.md` 里的角色和四格漫画固定设定。
2. 再追加本文件的公共设定。
3. 按画面主题追加一个或多个页面布局文件。
4. 最后追加版本主题 prompt，例如 `v4.1.0-ui-comic.prompt.md` 或 `v4.1.1-ui-comic.prompt.md`。

页面布局文件：

- `layout-00-app-shell.prompt.md`：桌面主框架、左侧导航、主内容区、全局抽屉。
- `layout-01-gallery.prompt.md`：画廊页、Provider 过滤树、分页器、图片网格、预览层、右键菜单、多选操作、空状态。
- `layout-03-albums.prompt.md`：画册列表、画册详情、子画册、面包屑、虚拟磁盘入口。
- `layout-04-plugins.prompt.md`：插件浏览页、插件详情页、插件卡片、商店源、安装配置。
- `layout-05-tasks-auto-configs.prompt.md`：任务详情、失败图片、任务抽屉、运行配置。
- `layout-06-settings-help.prompt.md`：设置页、帮助页、快捷键、说明内容。
- `layout-07-mobile-compact.prompt.md`：移动端 / 紧凑布局、底部 Tab、抽屉式操作。

通用视觉约束：

应用 UI 元素应该简化成可读的漫画背景，不要画成真实软件截图，也不要塞满无法识别的小字。左侧导航、顶部工具栏、图片网格、画册卡片、插件卡片这些结构要清楚。文字可以只保留短标签，例如“画廊”“画册”“插件”“设置”“尺寸”“格式”“名称”“4.1.1”“4.1.0”。如果图片模型不擅长文字，请让气泡和标签留白，后期再加字。

应用界面主导的格子最多两个，其他格子自由发挥。

顶部工具栏具体元素：

标题行的“共 2345 张”靠近页面标题，表示当前画廊总图片数。刷新按钮用圆形箭头图标；收集按钮表示网络爬取；导入按钮表示本地图片 / 视频 / 文件夹导入；快速设置按钮用齿轮图标；帮助按钮用问号图标。这些按钮可以是小型 Element Plus 风格按钮，不要画得很大。

浏览控制行具体元素：

筛选按钮（展开provider过滤树）可以带漏斗图标，排序按钮可以带上下箭头。分页器紧跟在筛选和排序后面，显示“‹ 1 / 24 ›”，左右箭头用于翻页。每页数量显示为“每页 100”或下拉选择。搜索框放在这一行偏右位置，提示文字是“搜索图片...”，用于按图片名称查找。

Provider 过滤树功能与故事：

筛选按钮展开的是 Kabegame 的 Provider 过滤树。它不是普通的单选筛选菜单，而是一棵“壁纸仓库路径树”：用户可以沿着不同路径浏览同一批壁纸，例如全部图片、按日期、按插件来源、按图片格式、按图片尺寸、按图片名称、按壁纸顺序。Provider 像应用背后的向导，把复杂数据整理成可点击的目录，让画廊、画册、任务详情和虚拟磁盘都能用相似的路径方式查看图片。

漫画里可以把 Provider 过滤树画成从“筛选”按钮长出来的发光小树或目录地图。树枝上挂着短标签：“全部”“日期”“插件”“格式”“尺寸”“名称”。看板娘像图书管理员一样顺着树枝给壁纸分类，小乌龟助手负责举路牌或盖章。故事感是“图片太多也不怕，Provider 会带路”。

Provider 过滤树视觉约束：

树要像轻量弹出层或可爱的目录地图，不能画成系统文件资源管理器。节点文字尽量短，最多保留“全部、日期、插件、格式、尺寸、名称”等关键词。如果图片模型容易乱写字，可以只画图标和空标签，后期再补文字。

图片网格具体元素：

网格卡片显示二次元壁纸缩略图。卡片角落可以有小标签：PNG、JPG、MP4、1920x1080、4K、竖屏、横屏。视频缩略图中央显示播放按钮，不要自动播放得很混乱。选中图片时卡片边框变成紫色，并出现小勾选标记。

图片预览结构：

双击图片后进入预览。预览是覆盖在 Kabegame 应用上的大图层，中央显示当前图片，左右有上一张 / 下一张箭头，左上或右上有关闭按钮，右上可以有“详情 i”入口。预览层不需要画出一排具体功能按钮，因为设置壁纸、加入画册、隐藏、删除等动作主要通过图片右键菜单或选择操作完成。预览画面要保留一点应用上下文，不能像脱离应用的纯图片查看器。

右键菜单与多选操作：

右键菜单是贴近图片卡片的小型浮层，菜单项可以用短标签表现：预览、收藏、加入画册、设置壁纸、隐藏、删除。漫画里只要画出菜单形状和几个短标签，不要画成复杂工具栏。

多选图片时，可以在画廊上方或网格附近出现选择操作条，显示“已选择 N 张”，并提供收藏、加入画册、设置壁纸、隐藏、删除等动作。也可以用更漫画化的方式表现：看板娘把多张图片用丝带捆起来，小乌龟助手举着“已选择”牌子。

分页器：

分页器属于第二层浏览控制行，不要单独画成很大的底部分页条。它应该轻量、紧凑，形式接近“‹ 1 / 24 ›”，旁边配合“每页 100”和搜索框。图片总数已经在标题行显示，不要在分页器里重复强调总数。

空状态：

画廊为空时，主内容区中央显示看板娘拿着空白图片卡，旁边是空相册盒子，按钮写“开始收集”或留白。台词可为“啊哦，这里还什么都没有呢”。

看板娘互动建议：

看板娘可以抱着一摞缩略图，把它们放进网格；也可以拿放大镜站在搜索框旁；可以把“PNG / JPG / 1920x1080”标签贴到图片角落；可以趴在预览大图边框旁检查细节；小乌龟助手可以推着分页器、举着播放按钮或递出右键菜单小牌子。

避免：

不要把画廊画成普通照片墙而没有工具栏。不要只画一个大图而没有网格。不要在预览层底部画一排具体功能按钮。不要让图片缩略图过于杂乱导致页面结构看不出来。不要出现过多不可读小字。

# 模糊滑杆的舞台监督
漫画主题：壁纸把应用也装饰了

为什么适合画：这是用户立刻能看见的视觉更新，适合画成壁纸从桌面跑进应用窗口。设置页、快速设置和画廊操作都能自然出镜，且可以用透明卡片和模糊滑杆做明确画面变化。

剧情角度：同一组应用内背景更新，演出角度偏反差吐槽：透明度和模糊像舞台灯光师一样调应用气质；落ち是调得太完美，看板娘开始检查自己是不是也被模糊了。

对应更新点：
- v4.1.0 新增前端本地的应用内背景图，默认启用，会把当前壁纸作为 Kabegame 应用背景，并提供开启/关闭、透明度和模糊设置。
- Settings 与 Quick Settings 暴露应用内背景相关开关、opacity、blur 控件；Settings cards 和 Plugin Browser cards 只在背景启用时变成透明卡片。
- Web 模式下图片动作改为 “Set as background”，把 currentWallpaperImageId 存进 localStorage，多选时使用第一张图片作为应用内背景。

```text
Kabegame 是一个基于 Tauri 的二次元壁纸爬取、管理、设置和轮播客户端。本漫画主题是 Kabegame v4.1.0 的应用内背景图和背景感知卡片：当前壁纸作为应用背景，设置页和快速设置可调开关、透明度、模糊；背景开启时设置卡片和插件卡片会呈现轻微透明效果。把更新画成“应用窗口拥有了自己的舞台灯光”。

版本更新点必须出现在画面中：v4.1.0、设置页、快速设置抽屉、应用内背景预览、透明卡片、模糊滑杆。可以出现短标签“背景”“透明”“模糊”“开”。

四格剧情：
第 1 格：设置页背景。看板娘站在 Kabegame 设置页中央，当前页面高亮“设置”，右侧是“应用设置”卡片。卡片后方还是普通浅色背景，她托腮，觉得壁纸管理器自己有点朴素。小乌龟助手抱着一张壁纸缩略图站在旁边。
第 2 格：快速设置抽屉从右侧滑出。看板娘把壁纸缩略图贴到应用窗口背后，抽屉里出现背景开关和两个滑杆。背景亮起但一开始太鲜明，插件卡片文字和缩略图有点抢画面；看板娘表情认真，像舞台监督在调灯。
第 3 格：她把透明度和模糊滑杆调到合适位置。画面显示设置卡片、插件卡片、画廊网格都在柔和背景上保持清晰，左侧导航仍然高亮清楚。小乌龟助手拿着小灯牌，把“刺眼”牌翻成“刚好”。
第 4 格：看板娘突然凑近镜面般的预览区，担心自己也被模糊掉，脸贴得很近；背景和卡片都很美，只有她的表情夸张变形。小乌龟助手淡定举牌“UI 清楚，人也清楚”。オチ是：调背景调到最后，开始怀疑自己是不是设置项的一部分。

对白建议：第 1 格：“壁纸工具也想有壁纸感。”第 2 格：“开灯！再柔一点！”第 3 格：“嗯，卡片也看得清。”第 4 格：“我没有被模糊吧？”

避免项：不要使用大面积纯渐变或装饰光球；不要让背景变成营销 hero；不要把设置控件写得密密麻麻；不要出现无法阅读的大段文字；不要把背景功能描述成真实跨平台桌面壁纸替代品，只表现应用内背景。
```
