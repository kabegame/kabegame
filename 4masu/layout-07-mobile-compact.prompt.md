Kabegame 移动端 / 紧凑布局 prompt：

用途：

当漫画需要表现 Android 预览版、手机界面、窄屏窗口或移动端操作时使用。

整体结构：

移动端不使用左侧导航栏，而是在底部使用 Tab 栏。主内容区从顶部到下方铺满屏幕，底部 Tab 固定。顶部工具栏更紧凑，很多操作会变成图标按钮或抽屉。

字符画草图：移动端画廊

```text
┌──────────────────────┐
│  画廊        [⚙] [?] │
│ [搜索...] [筛选]     │
│ [排序] [收集]        │
├──────────────────────┤
│ ┌────┐ ┌────┐        │
│ │图1 │ │图2 │        │
│ └────┘ └────┘        │
│ ┌────┐ ┌────┐        │
│ │图3 │ │图4 │        │
│ └────┘ └────┘        │
│                      │
├──────────────────────┤
│ 🖼画廊 📚画册 🧩插件 │
│ ⏰配置 ⚙设置         │
└──────────────────────┘
```

底部 Tab：

底部 Tab 包含画廊、画册、插件、运行配置、设置等入口。图标在上，文字在下。当前 Tab 用紫色高亮。由于空间有限，帮助可能通过顶部问号按钮或抽屉进入。

移动端画廊：

画廊网格通常是两列或三列。搜索、筛选、排序、收集等按钮更紧凑，可能换行排列。点击收集时出现底部选择器，让用户选择本地导入或网络收集。图片预览占满屏幕，左右滑动切换。

移动端画册：

画册卡片更小，通常两列。新建画册、刷新、快速设置可以放在顶部图标按钮中。画册详情仍有图片 / 子画册标签页，但更紧凑。

移动端插件：

插件卡片以图标为主，名称短，版本和状态标签可能横向滚动或简化显示。安装按钮在卡片底部。插件详情像手机设置页一样纵向滚动。

移动端抽屉：

快速设置、帮助、收集配置、任务信息经常使用底部抽屉或全屏弹窗。底部抽屉有圆角顶部，内容是表单、按钮和列表。

看板娘互动建议：

看板娘可以变成小尺寸 chibi 形象，坐在手机屏幕边缘；小乌龟壳背包精灵如果化为小乌龟助手，可以趴在底部 Tab 上，指向当前页面。不要让角色遮挡底部 Tab 和核心网格。

避免：

不要在移动端画左侧导航。不要把底部 Tab 漏掉。不要让按钮文字过长挤出屏幕。不要把移动端画成桌面窗口缩小版。
