Kabegame 插件页布局 prompt：

用途：

当漫画主题与插件商店、插件安装、插件自带 Provider 树、爬虫配置、插件源管理相关时使用。

插件浏览页结构：

左侧保留主导航。右侧顶部是插件页标题和操作按钮：刷新、导入插件源、管理源、快速设置、帮助。标题下方是标签页：已安装、官方源、其他插件源。主体是插件卡片网格。

插件卡片具体元素：

插件卡片包含图标、插件名称、简介、版本标签、已安装 / 未安装 / 可更新标签、安装 / 卸载 / 更新按钮。卡片整体应像软件插件商店，不要画成网页商品卡，也不要画成游戏角色卡。

插件详情页结构：

插件详情页顶部有返回按钮和插件标题。内容区显示插件图标、名称、版本、作者或来源、描述、文档图片、安装 / 卸载按钮。若表现运行配置，可以画参数输入框、下拉项、开关和“运行”按钮。

插件 Provider 树视觉：

如果主题是 4.1.0 插件自带 Provider 树，可以把每个插件卡片后面画出自己的小路径树或小房间。插件们把“按标签、按角色、按站点、按排行”等路径挂到 Kabegame 的画廊和虚拟磁盘入口下。

看板娘互动建议：

看板娘可以像店长一样站在插件卡片之间；给插件盖“已安装”印章；把插件小工具包插入 Kabegame；小乌龟壳背包精灵如果化为小乌龟助手，可以拿着“配置参数”表单或“插件源”地图。

避免：

不要把插件页画成浏览器扩展商店截图。不要让插件卡片全是不可读文字。不要画成只有一堆拼图图标而没有 Kabegame 应用框架。
