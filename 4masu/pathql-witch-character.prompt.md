PathQL 魔女角色设计 Prompt：

请设计一位 Kabegame 世界观中的“PathQL 路径魔女”角色。她是 Kabegame 壁纸魔法工坊里的路径魔法师，负责把复杂的壁纸数据整理成可以点击和浏览的路径树。她不是战斗魔女，而是可爱、聪明、理性、稍微有点小得意的魔法工程师 / 图书管理员型角色。

角色核心气质：

可爱但不幼稚，聪明、俏皮、可靠。她像在工坊里维护路径规则的前辈，能用魔法让画廊、画册、插件、任务详情和虚拟磁盘都按照同一套路径规则运转。表情可以是自信微笑、眨眼、轻轻托帽檐，像在说“路径已经安排好了”。

建议外观设计：

- 发色：建议使用深紫到靛蓝的渐变长发，发梢可以带一点青绿色或淡蓝色发光，和 Kabegame 看板娘的浅紫短发区分开，但仍属于紫粉系世界观。
- 眼睛：建议青绿色、蓝紫色或星空紫眼睛，瞳孔可以有小小的路径节点 / 星点高光。
- 服装：轻量魔女装，不要黑暗哥特。可以是紫黑色短斗篷、奶油白衬衫、深紫短裙或不对称长裙、细腰带、书袋、小披肩。衣服边缘带细线条路径纹样，像 UI 路径线或目录树。
- 帽子：小魔女帽或贝雷帽式魔法帽，帽檐上有路径节点、书签、斜杠 `/` 或小小的发光树枝装饰。不要做成巨大夸张法师帽。
- 道具：一本“PathQL”路径魔法书、发光卷轴、路标魔杖、目录树书签、悬浮的小路径节点。魔杖可以像一支细长的 UI 指针或羽毛笔。
- 配色：主色深紫、靛蓝、奶油白；点缀色青绿色或淡蓝；少量粉紫与 Kabegame 看板娘呼应。整体要清爽可读，不要全黑。
- 身高和体态：少女/年轻女性二次元角色，身形轻盈，动作优雅但带一点小得意。不要性感暴露，不要成熟黑暗魔女。

视觉符号：

她身边可以漂浮短路径符号和 UI 魔法元素，例如 `/gallery`、`/albums`、`/plugins`、`fetch`、`count`、`children`、发光节点、路径线、目录树、小传送门。符号要少而清楚，不能变成代码墙。Provider 过滤树可以像她召唤出的发光树，插件入口可以像挂在树枝上的小门牌。

与 Kabegame 看板娘的关系：

如果和 Kabegame 看板娘同框，PathQL 魔女负责施法整理路径，看板娘负责把整理好的壁纸放进画廊或画册。看板娘更元气治愈，PathQL 魔女更理性俏皮。两人的颜色要协调但能区分：看板娘浅紫粉，PathQL 魔女深紫蓝。

角色设计避免项：

不要画成黑暗反派，不要画成战斗魔法少女，不要巨大魔法阵，不要武器化，不要性感暴露，不要满屏代码，不要完全替代 Kabegame 看板娘成为主角。PathQL 魔女是功能拟人化辅助角色，登场时要服务于“路径、Provider、DSL、路由、分页、过滤树”等软件更新主题。
