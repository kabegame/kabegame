Provider 过滤器行功能与故事：

画廊工具栏有一个「筛选」切换按钮（带漏斗图标）。点击后，按钮后方展开一排横向的维度筛选标签按钮：「日期」「插件」「格式」「尺寸」「名称」「壁纸顺序」等，每个按钮对应一个筛选维度。点击某个维度按钮，会从按钮下方弹出一个轻量 popover，里面是该维度的 Provider 过滤树（例如日期维度展示「全部 / 今日 / 本周 / 本月」，插件维度展示「Pixiv / Konachan / …」）。用户在 popover 内选中节点后，该维度按钮变为激活状态（高亮），画廊实时刷新；多个维度可以同时激活，筛选条件叠加生效。

漫画里可以把筛选器行画成一排从「筛选」按钮后延伸出来的小标签胶囊，每个胶囊代表一个维度（日期、插件、格式……）；点击某个胶囊时，下方弹出一个可爱的小弹窗，里面是该维度的发光树节点。看板娘像分类管理员一样依次点击各维度胶囊，小乌龟壳背包精灵如果化为助手，可以举着对应维度的路牌。故事感是”筛选不是打开一棵大树，而是打开一排小门，每扇门后各有一片天地”。

筛选器行展开草图：


Provider 过滤器视觉约束：

筛选器行是工具栏里的一排横向小按钮，不是一棵展开的大树从页面中间弹出来。每个维度的 popover 弹窗小而精，里面树节点文字尽量短，最多保留”全部、今日、本周、Pixiv、PNG、横屏”等关键词。激活状态的维度按钮可以带浅色高亮或小圆点。如果图片模型容易乱写字，可以只画胶囊轮廓和小图标，后期再补文字。整体感是”一排维度胶囊 + 轻量 popover”，不是”一个展开的大型筛选面板”。

图片网格具体元素：

网格卡片显示二次元壁纸缩略图。卡片角落没有任何小标签，最多是视频缩略图的右上角有一个播放按钮。选中图片时卡片边框变成粉红色。
