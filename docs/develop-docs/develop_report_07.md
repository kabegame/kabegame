## 一、界面设计目标

Kabegame 的界面设计目标是让用户能够快速完成三个核心任务：

1. 通过插件获取素材。
2. 在画廊中浏览、筛选和整理素材。
3. 将画册或素材设置为壁纸并进行自动轮播。

界面设计需要兼顾普通用户和高级用户。普通用户应能通过直观按钮完成导入、浏览、设置壁纸等操作；高级用户则需要插件配置、任务日志、筛选排序和批量管理等能力。

因此，Kabegame 的界面不应只是一个图片展示页面，而应是围绕“收集 - 管理 - 使用”工作流设计的媒体管理工具。

## 二、设计原则

| 设计原则 | 说明 |
|---|---|
| 清晰性 | 页面入口明确，用户能够快速找到画廊、画册、插件、任务和设置 |
| 一致性 | 不同页面使用统一的按钮、弹窗、列表、卡片和状态提示 |
| 高效性 | 大量素材场景下支持分页、筛选、排序、多选和批量操作 |
| 可反馈性 | 导入、插件任务、壁纸设置等操作都应有进度、成功或失败提示 |
| 渐进展示 | 普通功能优先展示，高级配置放在详情页或设置页中 |
| 跨端适配 | 桌面端适合多栏布局和批量操作，移动端适合底部导航和触摸交互 |
| 容错性 | 插件失败、网络错误、空画廊、权限不足等情况应提供明确提示 |

## 三、整体界面结构

Kabegame 的整体界面采用“主导航 + 内容区域 + 状态反馈”的结构。

桌面端建议使用侧边栏或顶部导航组织主要功能，移动端则可使用底部导航。核心页面包括：

| 页面 | 主要作用 |
|---|---|
| 画廊页 | 浏览、筛选、排序、预览和批量管理素材 |
| 画册页 | 创建画册、整理素材、调整顺序、设置轮播来源 |
| 插件页 | 浏览插件、安装插件、配置插件、运行收集任务 |
| 任务页 | 查看导入或插件任务的进度、日志、结果和错误信息 |
| 壁纸设置页 | 设置壁纸模式、轮播来源、轮播方式和轮播间隔 |
| 设置页 | 配置存储路径、代理、语言、主题、权限和高级选项 |

整体信息结构如下：

```text
Kabegame
├── 画廊
│   ├── 全部素材
│   ├── 筛选 / 排序
│   ├── 图片预览
│   └── 批量操作
├── 画册
│   ├── 画册列表
│   ├── 画册详情
│   └── 轮播来源
├── 插件
│   ├── 插件商店
│   ├── 已安装插件
│   ├── 插件详情
│   └── 插件配置
├── 任务
│   ├── 运行中任务
│   ├── 已完成任务
│   └── 失败日志
└── 设置
    ├── 壁纸设置
    ├── 存储设置
    ├── 网络 / 代理
    └── 高级设置
```

## 四、画廊页界面设计

### 4.1 页面目标

画廊页是用户管理素材的主入口。用户应能在该页面完成浏览、预览、筛选、排序、多选、删除、加入画册和设置壁纸等操作。

### 4.2 页面布局

桌面端画廊页建议采用以下结构：

```text
┌──────────────────────────────────────────────┐
│ 顶部工具栏：导入 / 筛选 / 排序 / 批量操作 / 搜索 │
├───────────────┬──────────────────────────────┤
│ 左侧导航区     │ 图片 / 视频网格区域              │
│ - 全部素材     │ □ □ □ □ □ □                   │
│ - 日期筛选     │ □ □ □ □ □ □                   │
│ - 插件来源     │ □ □ □ □ □ □                   │
│ - 媒体类型     │                              │
├───────────────┴──────────────────────────────┤
│ 底部分页 / 当前数量 / 加载状态                  │
└──────────────────────────────────────────────┘
```

移动端画廊页应简化为：

```text
顶部：标题 + 搜索 / 筛选按钮
中部：双列或三列图片网格
底部：导航栏
弹出层：筛选、排序、批量操作
```

### 4.3 主要界面元素

| 元素 | 说明 |
|---|---|
| 导入按钮 | 用于导入本地图片、视频、文件夹或压缩包 |
| 筛选按钮 | 按插件来源、日期、媒体类型、画册、尺寸等筛选 |
| 排序按钮 | 按导入时间、名称、尺寸、壁纸设置时间等排序 |
| 搜索框 | 按文件名、来源或备注搜索 |
| 图片网格 | 展示素材缩略图，支持点击预览 |
| 多选模式 | 批量删除、加入画册、设置壁纸 |
| 分页控件 | 控制当前页和每页数量，避免大图库卡顿 |
| 空状态提示 | 当画廊为空时，引导用户导入素材或安装插件 |

### 4.4 交互说明

1. 用户点击缩略图，进入图片预览弹窗。
2. 用户右键或长按图片，显示快捷操作菜单。
3. 用户进入多选模式后，可以批量加入画册或删除。
4. 用户切换筛选条件后，图片网格应刷新并显示加载状态。
5. 用户导入素材后，新素材应出现在画廊中，并可继续加入画册。

### 4.5 状态设计

| 状态 | 界面表现 |
|---|---|
| 空画廊 | 显示“导入本地素材”和“安装插件获取素材”两个入口 |
| 加载中 | 显示骨架屏或加载提示，不阻塞其他页面 |
| 加载失败 | 显示失败原因和重试按钮 |
| 多选状态 | 缩略图显示选择框，底部或顶部显示批量操作栏 |
| 无筛选结果 | 提示当前筛选条件无结果，并提供清除筛选按钮 |

## 五、画册页界面设计

### 5.1 页面目标

画册页用于把零散素材组织成可复用集合。画册既是分类工具，也是壁纸轮播的重要来源。

### 5.2 页面布局

画册页分为画册列表和画册详情两层。

画册列表：

```text
┌──────────────────────────────┐
│ 顶部：新建画册 / 搜索画册       │
├──────────────────────────────┤
│ 画册卡片  画册卡片  画册卡片     │
│ 封面      名称      图片数量     │
└──────────────────────────────┘
```

画册详情：

```text
┌─────────────────────────────────────┐
│ 画册标题 / 描述 / 设置为轮播来源       │
├─────────────────────────────────────┤
│ 工具栏：添加图片 / 排序 / 删除 / 封面   │
├─────────────────────────────────────┤
│ 画册内图片网格                         │
└─────────────────────────────────────┘
```

### 5.3 主要界面元素

| 元素 | 说明 |
|---|---|
| 新建画册按钮 | 创建新的画册 |
| 画册卡片 | 展示封面、名称、数量和最近更新时间 |
| 画册详情头部 | 展示画册名称、描述、封面和操作按钮 |
| 设置为轮播来源 | 将当前画册用于壁纸轮播 |
| 排序控件 | 支持手动排序或按时间排序 |
| 添加图片入口 | 从画廊中选择素材加入画册 |

### 5.4 交互说明

1. 用户点击“新建画册”，输入名称后创建画册。
2. 用户从画廊多选图片后加入指定画册。
3. 用户进入画册详情后，可以调整图片顺序。
4. 用户可以将当前画册设置为壁纸轮播来源。
5. 用户可以修改画册封面，使画册列表更易识别。

### 5.5 状态设计

| 状态 | 界面表现 |
|---|---|
| 无画册 | 显示创建画册按钮和简单说明 |
| 空画册 | 显示“从画廊添加图片”入口 |
| 删除确认 | 删除画册前提示是否保留原始素材 |
| 设置轮播成功 | 显示成功提示，并提供跳转到壁纸设置入口 |

## 六、插件页界面设计

### 6.1 页面目标

插件页用于安装、管理和运行插件。它需要兼顾普通用户的“一键使用”和高级用户的“可配置、可排错”。

### 6.2 页面布局

插件页建议分为“插件商店”和“已安装插件”两个标签页。

```text
┌─────────────────────────────────────┐
│ 顶部：插件商店 / 已安装插件 / 导入插件   │
├─────────────────────────────────────┤
│ 插件卡片：名称 / 版本 / 描述 / 状态       │
│ 插件卡片：安装 / 更新 / 删除 / 运行       │
└─────────────────────────────────────┘
```

插件详情页：

```text
┌─────────────────────────────────────┐
│ 插件名称 / 版本 / 作者 / 状态           │
├─────────────────────────────────────┤
│ 插件说明                              │
├─────────────────────────────────────┤
│ 配置表单：页数 / 关键词 / 质量 / 分类     │
├─────────────────────────────────────┤
│ 操作按钮：开始收集 / 查看任务 / 删除插件   │
└─────────────────────────────────────┘
```

### 6.3 主要界面元素

| 元素 | 说明 |
|---|---|
| 插件卡片 | 展示插件名称、描述、版本、安装状态 |
| 导入插件按钮 | 从本地选择 `.kgpg` 插件包 |
| 安装 / 更新按钮 | 安装或更新插件 |
| 删除按钮 | 删除已安装插件 |
| 配置表单 | 根据插件声明动态生成配置项 |
| 开始收集按钮 | 使用当前配置启动插件任务 |
| 查看任务按钮 | 跳转到任务页查看进度和日志 |
| 网络提示 | 对需要代理的插件显示说明 |

### 6.4 交互说明

1. 用户进入插件页。
2. 用户从商店安装插件，或导入本地 `.kgpg` 文件。
3. 用户进入插件详情页查看说明。
4. 用户填写插件配置。
5. 用户点击“开始收集”。
6. 系统创建任务，并跳转或提示进入任务页。
7. 任务完成后，结果进入画廊。

### 6.5 状态设计

| 状态 | 界面表现 |
|---|---|
| 插件未安装 | 显示安装按钮 |
| 插件已安装 | 显示运行、更新、删除按钮 |
| 插件需要更新 | 显示版本提示和更新按钮 |
| 插件运行中 | 显示运行状态，并提供查看任务入口 |
| 插件失败 | 显示失败原因、日志入口和重试按钮 |
| 插件源不可用 | 显示网络错误或源不可达提示 |

## 七、任务页界面设计

### 7.1 页面目标

任务页用于展示插件收集、本地导入和其他长时间操作的执行状态。用户应能查看任务是否成功、失败原因是什么，以及任务产生了哪些素材。

### 7.2 页面布局

```text
┌─────────────────────────────────────┐
│ 顶部：全部 / 运行中 / 已完成 / 失败       │
├─────────────────────────────────────┤
│ 任务列表：名称 / 插件 / 进度 / 状态       │
├─────────────────────────────────────┤
│ 任务详情：日志 / 参数 / 结果图片          │
└─────────────────────────────────────┘
```

### 7.3 主要功能

| 功能 | 说明 |
|---|---|
| 查看进度 | 显示当前任务百分比或状态 |
| 停止任务 | 用户可以停止运行中的任务 |
| 查看日志 | 展示错误原因、请求失败、脚本异常等信息 |
| 查看结果 | 展示任务已经获取的图片或视频 |
| 加入画册 | 将任务结果加入指定画册 |
| 删除任务 | 清理历史任务记录 |

## 八、壁纸设置页界面设计

### 8.1 页面目标

壁纸设置页用于配置单张壁纸和自动轮播。用户应能清楚理解当前使用的壁纸来源、轮播方式和切换间隔。

### 8.2 页面布局

```text
┌─────────────────────────────────────┐
│ 当前壁纸预览                          │
├─────────────────────────────────────┤
│ 轮播开关：开启 / 关闭                  │
├─────────────────────────────────────┤
│ 轮播来源：选择画册                     │
├─────────────────────────────────────┤
│ 轮播方式：随机 / 顺序                  │
├─────────────────────────────────────┤
│ 切换间隔：分钟 / 小时                  │
├─────────────────────────────────────┤
│ 壁纸模式：原生模式 / 窗口模式            │
└─────────────────────────────────────┘
```

### 8.3 主要界面元素

| 元素 | 说明 |
|---|---|
| 当前壁纸预览 | 展示当前设置的壁纸缩略图 |
| 轮播开关 | 控制是否自动切换壁纸 |
| 轮播来源选择 | 选择一个或多个画册 |
| 轮播方式 | 随机或顺序 |
| 轮播间隔 | 设置自动切换时间 |
| 壁纸模式 | 原生模式、窗口模式或平台支持的其他模式 |
| 立即切换按钮 | 手动切换到下一张壁纸 |
| 暂停按钮 | 临时暂停轮播 |

### 8.4 交互说明

1. 用户从画册页点击“设置为轮播来源”，进入壁纸设置页。
2. 系统自动选中该画册作为轮播来源。
3. 用户选择随机或顺序轮播。
4. 用户设置切换间隔。
5. 用户开启轮播。
6. 系统显示当前壁纸和轮播状态。

### 8.5 状态设计

| 状态 | 界面表现 |
|---|---|
| 未选择轮播来源 | 提示用户选择画册 |
| 画册为空 | 提示该画册无可轮播素材 |
| 轮播运行中 | 显示下一次切换时间 |
| 轮播已暂停 | 显示暂停状态和恢复按钮 |
| 壁纸设置失败 | 显示失败原因和重试按钮 |
| 平台不支持某模式 | 禁用对应选项并显示说明 |

## 九、核心交互流程

### 9.1 插件收集素材流程

```text
插件页
→ 选择插件
→ 查看插件说明
→ 填写配置
→ 开始收集
→ 任务页查看进度
→ 收集结果进入画廊
→ 加入画册
```

### 9.2 画廊整理素材流程

```text
画廊页
→ 导入或筛选素材
→ 进入多选模式
→ 选择图片
→ 加入画册
→ 进入画册详情
→ 调整顺序或设置封面
```

### 9.3 壁纸轮播设置流程

```text
画册页
→ 选择画册
→ 设置为轮播来源
→ 进入壁纸设置页
→ 选择随机 / 顺序
→ 设置切换间隔
→ 开启轮播
```

## 十、异常状态与提示设计

| 场景 | 界面提示 | 建议操作 |
|---|---|---|
| 画廊为空 | “还没有素材，可以导入本地文件或安装插件获取素材” | 导入素材 / 前往插件页 |
| 插件安装失败 | “插件文件无效或版本不兼容” | 重新选择插件 / 查看说明 |
| 插件运行失败 | 显示失败原因和任务日志入口 | 查看日志 / 重试 |
| 网络不可用 | “网络连接失败，请检查网络或代理设置” | 前往代理设置 |
| 403 Forbidden | “目标来源拒绝访问，可能需要代理或插件更新” | 检查网络 / 更新插件 |
| 画册为空 | “当前画册没有素材” | 从画廊添加图片 |
| 壁纸设置失败 | “壁纸设置失败，请检查权限或平台支持情况” | 重试 / 查看设置 |
| 文件不存在 | “素材文件不存在，可能已被移动或删除” | 重新定位 / 从库中移除 |
| 大图库加载慢 | 显示加载进度和分页状态 | 调整每页数量 |

## 十一、跨端界面设计

| 平台 | 界面设计重点 |
|---|---|
| Windows / macOS / Linux | 适合侧边栏、多列画廊、批量操作、右键菜单和快捷键 |
| Android | 适合底部导航、双列图片网格、触摸预览和简化设置 |
| CLI | 不提供图形界面，但命令名称和功能逻辑应与 GUI 保持一致 |

跨端设计应遵循“信息结构一致，交互方式适配平台”的原则。例如桌面端可以使用右键菜单和多选工具栏，移动端则应使用长按、多选浮层和底部操作栏。

## 十二、界面验收标准

| 编号 | 验收标准 |
|---|---|
| UI-01 | 用户可以从主导航进入画廊、画册、插件、任务和设置页面 |
| UI-02 | 用户可以在画廊页导入素材并看到缩略图 |
| UI-03 | 用户可以在画廊中筛选、排序和预览图片 |
| UI-04 | 用户可以创建画册并从画廊加入图片 |
| UI-05 | 用户可以在插件页安装插件并启动收集任务 |
| UI-06 | 用户可以在任务页查看进度、结果和错误日志 |
| UI-07 | 用户可以从画册进入壁纸轮播设置 |
| UI-08 | 用户可以设置轮播来源、轮播方式和切换间隔 |
| UI-09 | 插件失败、网络失败、空画廊和壁纸设置失败时有明确提示 |
| UI-10 | 移动端界面不应出现明显文字溢出、按钮遮挡或无法点击的问题 |

## 十三、总结

Kabegame 的界面设计应围绕“收集、管理、使用”三个动作组织。插件页负责收集素材，画廊和画册负责管理素材，壁纸设置页负责使用素材。

界面设计的重点不是堆叠功能，而是让用户能够清楚理解当前处于哪个流程、下一步可以做什么、操作是否成功以及失败时如何处理。对于 Kabegame 这类跨平台、插件化、媒体管理类应用，清晰的信息结构和可靠的状态反馈比单纯的视觉装饰更重要。
