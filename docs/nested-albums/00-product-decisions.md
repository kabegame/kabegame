# 产品决策（嵌套画册）

以下语义已确定，实现时以此为准；若与旧文档或其它章节冲突，以本文件为准。

## 1. 图片挂载层级

**任意一层画册均可直接挂图。** 父画册可同时包含：

- 本层直接关联的图片；
- 子画册（作为子节点）。

「本层图片」与「子树内图片」在统计、轮播、去重等场景下需区分，见下文。

## 2. 「打开画册」的展示范围（前端与虚拟盘一致）

有两处入口：**应用内画册界面**与**虚拟盘**（在支持虚拟盘的平台）。

两者行为对齐：

- 当前画册下展示：
  - **子画册**（以文件夹形式呈现）；
  - **仅本层直接包含的图片**（不展示子孙画册里的图片）。
- **不包含嵌套聚合视图**：即打开某画册时，**不是**「把整棵子树所有图片打平列在一个列表里」；用户需进入子画册才能看到更深层的图片。

（若某处需要「整树图片数量」等元数据，可单独用计数 API 表达，与浏览列表语义分开。）

## 3. 画册预览图（封面条 / 缩略预览）

目标：用少量缩略图代表整本画册，**优先体现浅层**，不足时再向深层借图。

- **优先浅层**：先使用**当前画册本层直接关联图片**作为预览来源。
- **不足时向子画册借**：当预览图**少于 3 张**时：
  1. 为**每个子画册**获取其「预览图」集合（子画册自身也按同样规则递归定义：先本层、不足再向下一层借）；
  2. 将**仍缺的预览位**在子画册的预览图之间**均匀分配**抽取（从各子画册预览中取样），凑满目标数量或用尽可用图。

具体常量（例如目标是否为 3 张、子画册预览各自取几张）实现时可与 UI 设计对齐，但**算法意图**为：浅层优先 + 子画册接力 + 均匀分摊缺口。

## 4. 壁纸轮播

- 支持配置：**是否包含子画册**（递归合并子树中的图片参与轮播）。
- **默认：包含子画册**（开启递归）。

由于同一图片可能出现在父册与子册或多次出现在子树中，**递归展开参与轮播的集合后必须去重**（按图片 `image_id` 或项目内唯一标识），再进入随机/顺序逻辑。

## 5. 收藏夹（系统画册）

- 收藏夹**可以有子画册**，也可以作为**其它画册下的子节点**（嵌套在树中任意合法位置）。
- **不可删除**（与现有一致：系统保护）。
- **「直接收藏」语义不变**：用户执行收藏操作时，图片进入**收藏夹的浅层**（直接挂在收藏夹根/收藏夹本层），不自动塞进某个子画册；子画册由用户自行整理产生。
